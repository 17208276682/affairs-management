# 组织管理功能说明

## 一、页面概述

组织管理页面（`/org/dept`）用于管理公司的部门层级结构。页面分为左侧**部门结构树**和右侧**部门信息面板**。

仅管理员（admin）、CEO 和高级管理者（director）角色可访问此页面。

前端展示规则：管理员账号仅用于登录与权限控制，不在组织管理界面的成员信息中展示。

---

## 二、部门树

- 左侧以树形结构展示所有部门，默认全部展开
- 点击某个部门节点后，右侧显示该部门的详细信息
- 树中仅展示部门节点（不展示成员）
- 树标题统一为"部门结构"
- 树仅保留展开形态，不提供折叠功能
- 在"部门结构"标题同一行右侧增加"新增人员"按钮

---

## 三、部门信息展示

### 3.1 顶级节点（公司根节点，level = 0）

| 字段 | 展示规则 |
|------|----------|
| 部门名称 | 部门本身的名称（如 xxx公司） |
| 负责人 | **显示 CEO 的姓名**。从组织树中查找顶级部门下的成员获取 |
| 成员数 | **显示所有用户的总数量**。计算方式为全部部门 memberCount 之和 |
| 上级部门 | — （无上级） |

### 3.2 非顶级节点（level > 0）

| 字段 | 展示规则 |
|------|----------|
| 部门名称 | 部门本身的名称 |
| 负责人 | **自动检测该部门中非普通员工（director / manager）的姓名**。由后端自动从该部门成员中查找角色不为 staff 的人。若无非普通员工则为空 |
| 成员数 | 该部门的 memberCount（仅自身部门的人数，不含子部门） |
| 上级部门 | 上级部门名称 |

---

## 四、部门操作

### 4.1 新增下级部门

- 选中某部门后点击"新增下级"按钮
- 表单填写：部门名称（必填），上级部门（自动填入当前选中的部门，不可修改）
- 保存后自动刷新部门树，选中新创建的部门
- 新增下级默认排在同级已有部门的最后一位（末位追加）

### 4.2 编辑部门

- 仅修改部门名称
- 上级部门不可修改

### 4.3 删除部门

- **顶级部门不可删除**（按钮禁用）
- 删除非顶级部门时弹窗确认
- 删除将**级联删除**所有子部门，并清理相关的用户管理关系
- 若当前部门或任一下级部门存在成员，则禁止删除

---

## 五、下级部门列表

右侧面板底部展示当前部门的直接下级部门列表，包含：

| 列 | 说明 |
|----|------|
| 部门名称 | 子部门名称 |
| 负责人 | 子部门负责人（同上述规则） |
| 人数 | 子部门的 memberCount |
| 层级 | 子部门的 level |

列表内部启用滚动（包含表格内部纵向滚动与横向滚动），便于字段较多和记录较多时浏览。

---

## 六、本部门成员列表

右侧面板新增"本部门成员"表格，点击左侧部门节点后展示该部门成员：

| 列 | 说明 |
|----|------|
| 姓名 | 成员姓名 |
| 部门 | 所属部门名称 |
| 职位 | 岗位名称 |
| 手机号 | 联系电话 |
| 邮箱 | 邮箱地址 |
| 角色 | 角色标签 |
| 操作 | 编辑、删除 |

本部门成员表同样启用表格内部滚动。

本列表会过滤管理员账号，不显示管理员成员信息。

### 6.1 成员编辑（与人员管理对齐）

在组织管理页面点击本部门成员的"编辑"，弹窗字段与人员管理保持一致：

| 字段 | 说明 |
|------|------|
| 姓名 | 必填 |
| 重置密码 | 编辑时可选，开启后显示密码输入 |
| 部门 | 树形选择，支持调整所属部门 |
| 职位 | 必填 |
| 手机号 | 必填，且作为登录账号 |
| 邮箱 | 可选，需符合邮箱格式 |
| 角色 | 顶级部门强制 CEO，其他部门可选高级管理者/中级管理者/普通员工 |

新增成员同样复用该弹窗和校验规则（含强密码、顶级节点仅1人、同部门 manager 唯一性）。

## 七、技术实现

### 7.1 前端

- 页面文件：`frontend/src/views/organization/DeptManageView.vue`
- 使用 `displayLeaderName` computed 属性动态计算负责人展示名
- 使用 `displayMemberCount` computed 属性动态计算成员数展示值
- 部门树组件使用 Element Plus 的 `el-tree`
- 新增本部门成员表格（含编辑、删除操作），并在部门切换/增删改后刷新成员数据
- 下级部门表和本部门成员表均使用内部滚动容器
- 树组件设置为默认展开且不可折叠（隐藏折叠箭头并禁用节点点击展开/收起）
- 组织管理树标题右侧新增"新增人员"按钮，直接打开成员新增弹窗
- 成员新增/编辑弹窗字段与人员管理页面保持一致

### 7.2 后端

- 服务实现：`DeptServiceImpl.java`
- 组织树接口 `getOrgTree()`：一次性加载所有部门和用户，在内存中组装树
  - 非顶级部门：自动从该部门的用户中查找 role ≠ 'staff' 的人作为负责人
  - 顶级部门：通过 leaderId 查找负责人（即 CEO）
- 组织树装配时子节点改为末位追加，避免新节点显示在第一位
- 部门列表接口 `getDeptList()`：调用 `toDeptVO()` 转换
  - 非顶级部门：查询部门成员中非 staff 作为负责人
  - 顶级部门：按 leaderId 查找
- 新增 `selectMaxSortByParentId`：创建下级部门时默认末位排序
- 删除部门前增加成员校验：当前部门或下级部门存在成员时禁止删除

### 7.3 角色体系

| 角色代码 | 显示名称 | 说明 |
|----------|----------|------|
| admin | 管理员 | 系统管理员，固定账号（username: admin） |
| ceo | CEO | 公司最高管理者，位于顶级节点 |
| director | 高级管理者 | |
| manager | 中级管理者 | 每个部门最多一名 |
| staff | 普通员工 | |
