## 一、系统概述

本系统为中小企业提供数字化事务管理解决方案，涵盖事务下达、跟踪、执行、审批和统计分析的完整闭环。系统支持多角色权限体系，不同角色登录后看到的菜单和功能各不相同。

**系统访问地址**：部署后通过浏览器访问（推荐 Chrome / Edge 浏览器）

---

## 二、角色说明

系统共有 **四类角色**，每个角色拥有不同的菜单和操作权限：

| 角色 | 系统标识 | 显示名称 | 职责定位 |
|------|---------|---------|---------|
| 系统管理员 | `admin` | 管理员 | 维护组织架构、管理人员、查看全局事务统计 |
| 总经理 | `ceo` | 总经理 | 全局事务管控，向任意员工下达事务，查看全员统计 |
| 副总经理 | `director` | 副总经理 | 与总经理权限相同，属于高层管理者 |
| 负责人（部门经理） | `manager` | 负责人 | 管理本部门事务，接收上级任务并可向下分派，向管辖部门下达事务 |
| 普通员工 | `staff` | 普通员工 | 接收事务、执行任务、提交成果 |

> **说明**：总经理和副总经理在系统中统称为 **"总经办"**，两者权限一致。

###  登录系统

1. 连接**HUAWEI-C506-1**，打开浏览器，**访问系统地址**：http://192.168.3.16/

1. 在登录页点击「忘记密码」链接

3. 点击「获取验证码」，系统发送短信验证码

5. 输入 **确认新密码**

7. 点击「确认重置」

1. 点击右上角头像区域 → 选择「个人中心」
2. 切换到「修改密码」选项卡
3. 输入手机号 → 获取验证码 → 输入新密码 → 确认新密码 → 输入验证码
4. 点击「修改密码」

###  退出登录

点击右上角头像区域 → 选择「退出登录」

## 四、系统管理员操作指南

###  可见菜单

管理员登录后，左侧菜单栏显示以下三项：

| 菜单 | 说明 |
|------|------|
| 组织管理 | 维护部门架构，增删改查部门，管理部门成员 |
| 人员管理 | 管理全系统人员，新增/编辑/删除成员，分配角色 |
| 事务统计 | 查看全局事务统计数据、图表分析 |

> **注意**：管理员 **不能** 查看事务总览和事务列表页面，也不能创建或执行事务。

---

### 组织管理

####  查看部门

1. 点击左侧「组织管理」菜单
2. 左侧面板显示 **部门树形结构**，所有层级部门自动展开
3. 点击某个部门节点，右侧显示该部门的详细信息：
   - 部门名称、部门ID、层级、上级部门、负责人、成员数
   - **下级部门列表**（含名称、负责人、人数、层级）
   - **本部门成员列表**（含姓名、部门、职位、手机号、邮箱、角色、操作按钮）

####  新增下级部门

1. 在左侧部门树中选中一个部门
2. 点击右侧面板的「新增下级」按钮
3. 在弹出的对话框中输入 **部门名称**
4. 上级部门自动填充为当前选中的部门
5. 点击「保存」

####  编辑部门

1. 选中目标部门
2. 点击「编辑」按钮
3. 修改部门名称
4. 点击「保存」

####  删除部门

1. 选中目标部门
2. 点击「删除」按钮（顶级部门不可删除）
3. 确认对话框提示「确认删除该部门及其全部下级部门？」
4. 点击「确认」执行删除

> **注意**：删除部门将 **级联删除** 所有下级子部门，请谨慎操作。

####  在组织管理中新增/编辑成员

1. 在部门树中选中部门后，点击左上方「新增人员」按钮
2. 弹出表单对话框，填写以下信息：
   - **姓名**（必填）
   - **密码**（新增时必填，编辑时可选择是否重置密码）
   - **部门**（下拉树选择）
   - **职位**（必填）
   - **手机号**（必填，唯一）
   - **邮箱**
   - **角色**（根据所选部门自动过滤可选角色）
   - **兼任负责人**（仅总经理/副总经理可见，可多选第二层部门）
3. 点击「保存」

编辑成员：在成员列表中点击「编辑」按钮，弹出相同表单进行修改。
删除成员：在成员列表中点击「删除」按钮，确认后删除。

### 人员管理

####  查看人员列表

1. 点击左侧「人员管理」菜单
2. 左侧面板显示 **部门树**，可点击部门筛选该部门下的人员
3. 右侧为人员列表，展示：姓名、部门、职位、手机号、邮箱、角色
4. 支持多维度筛选：
   - **按姓名搜索**
   - **按部门筛选**（下拉选择）
   - **按职位搜索**
   - **按角色筛选**（总经理/副总经理/负责人/普通员工）
5. 点击「重置」清除所有筛选条件

####  新增人员

1. 点击页面右上角「新增人员」按钮
2. 填写表单信息（同 4.2.5 节）
3. 点击「保存」

####  编辑人员

1. 在人员列表中找到目标人员
2. 点击该行「编辑」按钮
3. 修改信息后点击「保存」
4. 编辑时可开启「重置密码」开关来重设密码，也可点击「随机生成」生成强密码

####  删除人员

1. 点击该行「删除」按钮
2. 确认后执行删除

####  管理部门（人员管理页内）

人员管理页面的左侧部门树同样支持部门操作：
- 选中部门后可点击「新增下级」创建子部门
- 点击「编辑」修改部门名称
- 点击「删除」删除部门（顶级不可删除）

####  角色分配规则

- **顶级部门**（总经办, level=0）：可分配总经理、副总经理
- **二级部门**（level=1）：可分配负责人、普通员工
- **三级及以下部门**：可分配普通员工和负责人
- **总经理/副总经理** 可 **兼任** 多个二级部门的负责人（通过"兼任负责人"多选框设置）

###  事务统计

####  进入统计页面

点击左侧「事务统计」菜单，页面分为以下五个区域：

####  事务状态卡片

页面顶部展示整体事务状态概览：
- **下达事务数**（总计，占比100%）
- **按时完成数** — 在截止日期前完成并审核通过
- **逾期完成数** — 超过截止日期后完成并审核通过
- **审核未通过** — 被驳回的事务数
- **待办事务数** — 未到截止时间且未完成
- **逾期未完成** — 超过截止时间仍未完成
- **作废事务数** — 已取消的事务

每个卡片显示 **数量** 和 **占比百分比**。

####  事务级别卡片

展示 A/B/C/D 四个级别的事务分析：
- 每个级别显示：下达数、完成数、未通过数、待办数
- 同时显示 **平均完成时长**（小时）和 **平均逾期时长**（小时）

| 级别 | 含义 |
|------|------|
| A — 紧急重要 | 重要且紧急的事务 |
| B — 紧急不重要 | 不重要但紧急的事务 |
| C — 不紧急重要 | 重要但不紧急的事务 |
| D — 不紧急不重要 | 不重要也不紧急的事务 |

####  部门事务图表

横向堆叠柱状图，按 **部门** 维度展示：
- 横轴表示事务数量
- 纵轴为各部门名称
- 堆叠颜色分别代表：按时完成（绿）、逾期完成（橙）、审核未通过（红）、待办事务（蓝）、逾期未完成（紫）、已作废（灰）
- 数据超过 8 个部门时自动出现滚动条

####  人员事务图表

横向堆叠柱状图，按 **人员** 维度展示（格式同部门事务图表）

####  每日事务折线图

左侧折线图，展示选定月份中每天各类事务数量的趋势变化：
- 可通过顶部按钮切换：「上月」「本月」「次月」或使用日期选择器自定义月份
- 6 条折线分别对应 6 种事务分类

####  每月事务折线图

右侧折线图，展示选定年份中每月各类事务数量的趋势变化：
- 可通过顶部按钮切换：「上年」「本年」「次年」或使用年份选择器自定义
- 6 条折线分别对应 6 种事务分类

**四象限视图**：

- 每个象限显示事务计数和列表

###  事务列表

###  创建事务（下达事务）

1. 在「我的下达」或「事务列表」页面，点击右上方「新建事务」按钮
2. 右侧面板切换为创建表单，填写以下信息：

| 字段 | 说明 | 要求 |
|------|------|------|
| 事务描述 | 事务的详细内容 | 必填，最多 2000 字 |
| 事务紧急性 | 选择四象限类型 | 必填，从重要紧急/重要不紧急/不重要紧急/不重要不紧急中选择 |
| 截止时间 | 完成截止日期 | 必填，提供快捷选项（半小时/一小时/一天/一周）或自定义 |
| 执行人 | 选择执行该事务的人员 | 必填，点击「选择执行人」弹出人员选择窗口 |
| 附件 | 相关文件 | 可选，最多 10 个文件 |

3. 点击「下达事务」按钮完成创建
4. 执行人将收到系统通知

**选择执行人**：
- 点击「选择执行人」按钮
- 在弹出的人员选择窗口中，可通过部门树筛选 + 姓名搜索
- 总经办可向 **任意员工** 下达事务
- 点击人员即选中，显示姓名和部门

---

###  查看事务详情

在列表中点击事务行或「详情」按钮，进入事务详情页（或右侧面板）：

**内联面板**（右侧）：
- 事务描述、附件（支持预览/下载）
- **流程动态**：时间线形式展示事务全部操作记录（创建 → 接收 → 处理 → 提交 → 审批等）
- 任务信息：下达人、下达部门、执行人、截止时间

**独立详情页**：
- 左侧主内容：事务描述、附件列表（支持在线预览和下载）、处理记录时间线
- 右侧信息栏：下达人信息、执行人信息、时间信息（创建/响应截止/实际响应/完成截止/实际完成/剩余时间）、事务级别
- 操作按钮区：
  - 若当前用户是执行人且事务未完成 → 显示「接收并执行」/「继续执行」
  - 若当前用户是下达人且事务状态为"已提交" → 显示「审核反馈」

---

###  审核反馈

当执行人提交成果后，事务状态变为"已提交"，下达人可以进行审核：

1. 在「我的下达」中找到状态为"待审核"的事务，点击进入详情
2. 点击「审核反馈」按钮，进入反馈页面
3. 页面展示执行人提交的成果内容和附件
4. 选择操作：

**通过**：
- 切换到「通过」标签页
- 可填写审核意见（可选）
- 点击「审核通过」→ 事务标记为已完成

**驳回**：
- 切换到「驳回」标签页
- 必须填写 **驳回理由**
- 点击「驳回」→ 事务退回给执行人，执行人可修改后重新提交

**转派**：
- 切换到「转派」标签页
- 点击「选择执行人」选择新的执行人
- 可填写转派说明
- 点击「确认转派」→ 事务转给新执行人处理

###  作为执行人接收和执行事务

总经办人员也可能被其他人下达事务（如其他管理层），此时在「我的代办」中处理：

1. 在「我的代办」中找到状态为"待处理"的事务
2. 点击事务行，右侧面板显示简化详情（隐藏部分任务信息）
3. 通过详情页的「接收并执行」→ 进入执行页面
4. 执行流程详见 [七、普通员工操作指南 - 7.3 执行事务](#73-执行事务)

## 六、负责人（部门经理）操作指南

### 可见菜单

负责人登录后，左侧菜单栏显示：

| 菜单 | 说明 |
|------|------|
| 事务总览 | 事务看板，分为"上级事务"和"部门事务"两栏统计 |
| 事务列表 | 查看管辖范围内事务 |
| 我的下达 | 查看自己下达给部门成员的事务 |
| 我的代办 | 查看上级下达给自己的事务 |

---

### 事务总览（Dashboard）

负责人的事务总览与总经办有所不同，特别在以下方面：

####  双栏状态卡片

页面左上方显示 **两组状态卡片**（上下排列）：

**上级事务**（收到的事务统计）：
- 按时完成数、逾期完成数、审核未通过、待办事务数、逾期未完成、作废事务数
- 右上角显示「事务总数：N」

**部门事务**（下发给部门的事务统计）：
- 相同的六项统计指标
- 右上角显示「事务总数：N」

#### 事务动态（带筛选）

支持三种筛选：
- **全部**：显示所有动态
- **上级**：仅显示与上级事务相关的动态
- **部门**：仅显示与部门事务相关的动态

####  事务级别饼图（带筛选）

支持三种筛选：
- **全部**：显示上级+部门所有事务的级别分布
- **上级**：仅统计上级下发的事务
- **部门**：仅统计自己下发给部门的事务

####  事务统计图表（双模式切换）

底部事务统计图表有两个模式，通过右上角的单选按钮切换：

- **部门事务**（默认）：横向堆叠柱状图，按部门成员展示事务状态分布
- **上级事务**：折线图，展示选定月份每日事务趋势
  - 顶部切换：「上月」「本月」「次月」或自定义月份

---

###  创建事务（下达事务）

1. 在「我的下达」或「事务列表」页面，点击「新建事务」
2. 填写事务信息（同总经办创建流程）
3. **执行人范围**：负责人只能选择 **管辖部门内的成员** 作为执行人

---

###  接收上级事务

负责人可能收到总经办下达的事务：

1. 在「我的代办」中查看上级下达的事务
2. 点击事务行查看详情

**负责人收到的事务有特殊处理流程**（内联操作）：

- **直接执行**：自己接收并执行任务，添加处理记录后提交
- **向下分派（转交）**：
  1. 在事务详情面板中，可选择「转交给下级」
  2. 选择部门内的成员作为实际执行人
  3. 填写补充说明
  4. 点击「确认下发」
  5. 系统创建子任务分配给选中的成员
  6. 面板状态变为「已转交，等待下级提交」

**下级提交后的流程**：
- 当下级成员提交成果后，面板显示下级提交的内容和附件
- 负责人可查看后点击「提交给上级」，将成果汇总提交
- 也可补充自己的内容后再提交

---

###  审核部门事务

当部门成员提交事务成果后：

1. 在事务列表或我的下达中找到状态为"待审核"的事务
2. 进入详情后点击「审核反馈」
3. 操作选项（通过/驳回/转派）同总经办审核流程

---

###  任务全景追踪

右侧面板与总经办相同，提供日视图、三日视图、周视图、月视图、四象限视图。

---

## 七、普通员工操作指南

###  可见菜单

普通员工登录后，左侧菜单栏显示：

| 菜单 | 说明 |
|------|------|
| 事务总览 | 个人事务看板和统计 |
| 事务列表 | 查看管辖范围内事务 |
| 我的代办 | 查看分配给自己的所有待办事务 |

> **注意**：普通员工 **没有**「我的下达」菜单，不能创建或下达事务。

---

###  事务总览（Dashboard）

####  状态卡片

页面左上方显示 **6 个状态卡片**：
- 按时完成数、逾期完成数、审核未通过、待办事务数、逾期未完成、作废事务数

####  事务动态

显示与自己相关的事务操作动态。

####  事务级别饼图

展示自己收到的事务在 A/B/C/D 四个级别的分布。

####  事务统计折线图

折线图展示每日事务趋势，可切换月份（上月/本月/次月/自定义）。

####  任务全景追踪

右侧面板与其他角色相同，提供日/三日/周/月/四象限五种视图。

---

###  执行事务

####  接收事务

1. 在「我的代办」中找到状态为 **"待处理"** 的新事务
2. 点击事务行查看详情，或点击「详情」进入独立详情页
3. 点击「接收并执行」按钮
4. 进入执行页面，页面顶部显示任务摘要（事务级别、标题、下达人、截止时间、状态）
5. 点击「确认接收」按钮 → 事务状态变为"已接收"，系统开始计时

####  添加处理记录

接收事务后，可以多次添加处理记录来记录工作进展：

1. 在执行页面的「添加处理记录」区域
2. 填写 **处理内容**（描述进展、遇到的问题等）
3. 可上传 **附件**（最多 3 个文件，支持预览/下载/取消）
4. 点击「提交记录」
5. 提交的记录会显示在右侧时间线中

> **说明**：首次添加处理记录后，状态自动变为"执行中"。

####  提交成果

执行完毕后，提交成果等待审核：

1. 在执行页面的「提交成果」区域
2. 填写 **完成说明**（描述成果情况）
3. 上传 **成果附件**（最多 5 个文件）
4. 点击「提交审核」
5. 系统弹出确认对话框「提交后将进入审核环节，是否确认提交？」
6. 确认后 → 事务状态变为"已提交"，等待下达人审核

####  被驳回后重新提交

如果下达人驳回了提交：
1. 事务状态变回"不通过"
2. 在「我的代办」中找到该事务
3. 点击「继续执行」→ 进入执行页面
4. 可继续添加处理记录或修改后重新提交

---

###  查看事务详情

在事务列表或我的代办中，点击事务行或「详情」按钮查看：
- 事务描述、附件预览和下载
- 流程动态时间线（所有操作记录）
- 任务信息（下达人、执行人、时间信息、级别）

---

### 附件操作

系统中所有附件均支持以下操作：
- **预览**：点击「预览」按钮，支持在线预览图片（PNG/JPG/GIF/WebP/SVG）、PDF、文本文件（TXT/MD/JSON/XML/CSV/HTML）、视频（MP4/WebM）、音频（MP3/WAV）
- **下载**：点击「下载」按钮，浏览器下载文件到本地

---

## 八、公共功能说明

###  个人中心

所有角色均可访问个人中心：

1. 点击右上角头像区域 → 选择「个人中心」
2. **左侧**：显示个人头像（名字首字）、姓名、角色标签
3. 下方信息列表：部门、职位、手机、邮箱
4. **右侧**：
   - 「基本信息」标签页：只读展示账号、姓名、角色、部门、职位、手机号、邮箱、状态
   - 「修改密码」标签页：通过手机号 + 短信验证码修改密码

> **管理员**的个人中心仅显示头像，部门/职位/手机/邮箱等信息显示为"-"。

###  通知消息

系统在事务状态变更时自动发送站内通知，通知类型包括：
- **事务分配**：事务被分配给您
- **事务接收**：您下达的事务已被执行人接收
- **成果提交**：执行人已提交成果，等待审核
- **审核通过**：事务审核通过
- **审核不通过**：事务被驳回
- **事务完成**：事务已完成

###  面包屑导航

页面顶部显示面包屑导航路径，方便定位当前所在页面位置。

###  侧边栏折叠

点击顶部栏左侧的折叠图标，可以折叠/展开左侧菜单栏，扩大工作区域。

## 九、事务状态流转说明

```
┌─────────┐
│  创 建   │  下达人创建事务
└────┬────┘
     ▼
┌─────────┐
│ 待接收   │  pending — 等待执行人确认接收
└────┬────┘
     ▼
┌─────────┐
│ 已接收   │  accepted — 执行人已确认，开始计时
└────┬────┘
     ▼
┌─────────┐
│ 执行中   │  in_progress — 首次添加处理记录后自动切换
└────┬────┘
     ▼
┌─────────┐
│ 已提交   │  submitted — 执行人提交成果，等待审核
└────┬────┘
     ├──────────────────┐
     ▼                  ▼
┌─────────┐      ┌─────────┐
│ 通  过   │      │ 不通过   │  rejected — 可修改后重新提交
│ approved │      └────┬────┘
└────┬────┘           │
     ▼                ▼
┌─────────┐    （回到执行中，重新提交）
│ 已完成   │
│completed │
└─────────┘

任意阶段可操作：
 ┌──────────┐
 │ 已取消    │  cancelled — 下达人可随时取消
 └──────────┘
 ┌──────────┐
 │ 已逾期    │  overdue — 系统自动标记超时
 └──────────┘
```

**关键规则**：
- 只有 **下达人** 可以取消事务
- 只有 **下达人** 可以审核通过/驳回
- 只有 **执行人** 可以接收、添加记录、提交成果
- 驳回后，执行人可修改后 **重新提交**
- 审核通过后，事务流程结束
- 负责人转交的事务，子任务状态与父任务联动

## 十、四象限优先级模型说明

系统采用"重要-紧急"四象限模型自动计算事务优先级：

| 象限 | 级别 | 重要性 | 紧急度 | 处理建议 |
|------|------|--------|--------|---------|
| 重要且紧急 | **A** | 高 | 高 | 立即处理，最高优先级 |
| 不重要但紧急 | **B** | 低 | 高 | 尽快处理，注意时间限制 |
| 重要但不紧急 | **C** | 高 | 低 | 计划处理，做好规划 |
| 不重要不紧急 | **D** | 低 | 低 | 可延后或委派 |

下达事务时，选择"事务紧急性"（四选一），系统自动标记对应的 A/B/C/D 级别。

## 十一、常见问题（FAQ）

### Q1：登录后页面空白或跳转异常？
**A**：清除浏览器缓存后重试。确保使用 Chrome 或 Edge 浏览器。

### Q5：负责人如何将收到的事务转交给下属？
**A**：在事务详情面板中选择「转交给下级」，选择部门成员后确认，系统自动创建子任务。

### Q6：管理员如何重置某个用户的密码？
**A**：在「人员管理」或「组织管理」中找到该用户，点击「编辑」，开启"重置密码"开关，输入新密码后保存。也可点击「随机生成」自动生成强密码。

### Q7：如何查看某个事务的完整处理历史？
**A**：进入事务详情页，右侧面板或独立详情页中的「处理记录」/「流程动态」展示了从创建到当前的所有操作记录。

### Q8：附件支持哪些格式预览？
**A**：支持以下格式的在线预览：
- 图片：PNG, JPG, JPEG, GIF, BMP, WebP, SVG
- 文档：PDF
- 文本：TXT, MD, JSON, XML, CSV, LOG, HTML
- 视频：MP4, WebM, OGG, MOV
- 音频：MP3, WAV, OGG, M4A, AAC
- 其他格式只支持下载

### Q9：事务逾期后会怎样？
**A**：系统自动标记超过截止时间的事务为逾期状态。逾期事务仍可继续执行和提交，但会被统计为"逾期完成"或"逾期未完成"。

### Q10：总经办和负责人的可下达范围有什么区别？
**A**：总经办（总经理/副总经理）可向 **任意员工** 下达事务；负责人只能向 **管辖部门内的成员** 下达事务。
