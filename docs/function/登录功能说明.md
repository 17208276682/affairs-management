# 登录功能说明

## 一、登录界面概述

登录页面采用左右分栏布局：
- **左侧品牌区**：展示系统名称"中小企业事务管理数智化系统"，标题下方以标签形式展示五个核心关键词：**事务列表**、**事务下达**、**事务代办**、**事务统计**、**组织管理**。下方配有品牌视觉动画场景。
- **右侧登录区**：包含登录表单和密码修改表单，两个界面独立展示（通过"忘记密码"按钮切换），各自拥有独立的标题区域。

---

## 二、登录表单

### 2.1 界面标题

登录表单显示独立标题区域：
- 图标：Monitor（蓝色显示器图标）
- 主标题：**欢迎登录**
- 副标题：Task Management System

### 2.2 输入字段

| 字段 | 说明 |
|------|------|
| 手机号 | 统一使用手机号登录 |
| 密码 | 用户密码 |
| 验证码 | 前端生成的4位随机字母数字验证码，不区分大小写 |

### 2.3 登录方式

系统前端统一为**手机号 + 密码**登录。后端仍兼容用户名查找逻辑，但业务侧不再暴露用户名输入。

### 2.4 验证码功能

- 验证码为4位随机字母数字组合（大写显示）
- 点击验证码区域可刷新生成新验证码
- 提交登录时校验验证码是否正确（不区分大小写）
- 验证码错误时提示"验证码不正确"并自动刷新
- 登录失败时自动刷新验证码

### 2.5 表单验证规则

| 字段 | 验证规则 |
|------|----------|
| 手机号 | 必填，格式为 `1[3-9]` 开头的11位数字 |
| 密码 | 必填，不少于4位 |
| 验证码 | 必填，且必须与当前显示的验证码一致（不区分大小写） |

### 2.6 记住密码

勾选"记住密码"后，登录成功时系统会将手机号和密码保存到浏览器 `localStorage`（key: `tm_remember`）。下次打开登录页时自动填充手机号和密码，并自动勾选"记住密码"。取消勾选后登录，将清除已保存的信息。

### 2.7 登录成功跳转

- **管理员（admin）/ CEO（ceo）**：默认跳转至 `/org/dept`（部门管理页）
- **其他角色**：默认跳转至 `/dashboard`（事务总览页）
- 如果 URL 中带有 `redirect` 参数，优先跳转至该地址

## 三、密码修改（忘记密码）

### 3.1 界面标题

点击登录表单中的"忘记密码"按钮，切换至密码修改界面。登录表单的"欢迎登录"标题会隐藏，显示密码修改的独立标题区域：
- 图标：Lock（蓝色锁图标）
- 主标题：**密码修改**
- 副标题：Reset Password

### 3.2 输入字段

| 字段 | 说明 |
|------|------|
| 手机号 | 用户注册时绑定的手机号 |
| 新密码 | 需满足密码强度要求（见下方规则） |
| 确认密码 | 再次输入新密码，必须与新密码一致 |
| 短信验证码 | 6位数字验证码（模拟发送） |

### 3.3 表单验证规则

| 字段 | 验证规则 |
|------|----------|
| 手机号 | 必填，格式为 `1[3-9]` 开头的11位数字 |
| 新密码 | 必填，8-20位，必须同时包含大写字母、小写字母、数字和特殊字符 |
| 确认密码 | 必填，且必须与新密码一致（实时校验） |
| 短信验证码 | 必填，固定6位数字 |

### 3.4 短信验证码

- 点击"获取验证码"按钮后，系统模拟发送6位数字短信验证码
- 验证码在页面上以提示文字显示（模拟短信）
- 提交时校验输入的验证码是否与发送的一致

### 3.5 操作按钮

- **返回登录**：关闭密码修改界面，回到登录表单
- **确认重置**：提交密码修改请求，成功后自动返回登录表单

## 四、密码强度要求

系统对新设置/修改的密码统一要求满足以下强度规则：

| 项目 | 要求 |
|------|------|
| 长度 | 8-20位 |
| 大写字母 | 至少包含1个（A-Z） |
| 小写字母 | 至少包含1个（a-z） |
| 数字 | 至少包含1个（0-9） |
| 特殊字符 | 至少包含1个（如 `@#$%^&*` 等非字母数字字符） |

正则表达式：`^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[^A-Za-z\d]).{8,20}$`

**校验位置：**
- 前端：密码修改表单的"新密码"字段
- 前端：人员管理中创建/编辑成员的"密码"字段
- 后端：`/auth/reset-password` 接口
- 后端：`/auth/change-password` 接口

> 注意：登录表单的密码字段不做强度校验，仅要求非空且不少于4位，以兼容已有账户。

## 六、技术实现

### 6.1 前端

- **框架**：Vue 3 + Element Plus + Pinia
- **登录页组件**：`frontend/src/views/login/LoginView.vue`
- **表单验证规则**：`frontend/src/utils/validate.ts`
- **用户API**：`frontend/src/api/user.ts`
- **认证工具**：`frontend/src/utils/auth.ts`（Token 存储/读取）
- **路由守卫**：`frontend/src/router/index.ts`（未登录自动跳转登录页）
- **记住密码**：使用 `localStorage` 存储，key 为 `tm_remember`

### 6.2 后端

- **框架**：Spring Boot + Spring Security + JWT + MyBatis
- **认证控制器**：`AuthController.java`（`/auth/login`、`/auth/reset-password`、`/auth/change-password`）
- **认证服务**：`AuthServiceImpl.java`（登录验证、密码重置、密码修改、密码强度校验）
- **用户详情服务**：`CustomUserDetailsService.java`（支持用户名和手机号查找用户）
- **密码编码器**：`NoOpPasswordEncoder`（明文存储，不加密）
- **密码强度校验**：后端 `resetPassword` 和 `changePassword` 均校验密码强度
- **JWT**：HS256签名，24小时有效期
- **安全配置**：`SecurityConfig.java`（CSRF禁用、无状态Session、JWT过滤）

### 6.3 接口说明

| 接口 | 方法 | 说明 | 认证 |
|------|------|------|------|
| `/auth/login` | POST | 用户登录（手机号+密码） | 否 |
| `/auth/userinfo` | GET | 获取当前用户信息 | 是 |
| `/auth/reset-password` | POST | 通过手机号重置密码（含密码强度校验） | 否 |
| `/auth/change-password` | POST | 修改密码（需验证旧密码+密码强度校验） | 是 |
