# 人员管理功能说明

## 一、页面概述

人员管理页面（`/org/member`）用于管理公司的所有人员信息。页面分为左侧**部门结构树**和右侧**人员列表 + 筛选**。

当前仅管理员（admin）可访问此页面。

前端展示规则：管理员账号仅用于登录与权限控制，不在人员管理列表及组织相关展示中出现。

---

## 二、部门结构树

- 左侧以树形结构展示所有部门，默认全部展开
- 点击部门节点后，右侧仅展示该部门的成员
- **点击顶级节点**（level = 0）时，展示**所有人员**
- 树顶部增加操作按钮：新增下级、编辑、删除（删除受部门成员约束）
- 树不提供折叠功能，始终保持展开
- 新增下级部门默认排在同级末尾

---

## 三、人员列表

### 3.1 展示列

| 列名 | 说明 |
|------|------|
| 姓名 | 用户真实姓名 |
| 部门 | 所属部门名称 |
| 职位 | 岗位名称 |
| 手机号 | 联系电话 |
| 邮箱 | 电子邮箱 |
| 角色 | 角色标签（总经理 / 副总经理 / 负责人 / 普通员工） |
| 操作 | 编辑、删除 |

说明：管理员账号不会出现在该列表中。

### 3.2 角色标签颜色

| 角色 | 标签类型 | 颜色 |
|------|----------|------|
| 总经理 | danger | 红色 |
| 副总经理 | danger | 红色 |
| 负责人 | warning | 橙色 |
| 普通员工 | info | 灰色 |

### 3.3 排序规则

- 列表支持在"姓名"、"部门"、"角色"三列点击排序箭头进行升降序排序
- 排序箭头显示在列标题右侧

人员列表按以下优先级排序：

1. **角色优先级**：总经理 > 副总经理 > 负责人 > 普通员工
2. **部门层级**：层级低的（靠近顶级）排在前
3. **部门名称**：中文排序
4. **姓名**：中文排序

---

## 四、筛选功能

页面顶部提供 4 个筛选条件，均可独立使用：

| 筛选项 | 类型 | 说明 |
|--------|------|------|
| 按姓名筛选 | 输入框 | 模糊匹配姓名 |
| 按部门筛选 | 下拉选择 | 选择指定部门 |
| 按职位筛选 | 输入框 | 模糊匹配职位名称 |
| 按角色筛选 | 下拉选择 | 总经理 / 副总经理 / 负责人 / 普通员工 |

点击"重置"按钮可一键清除所有筛选条件。

---

## 五、顶级节点特殊规则

### 5.1 顶层角色规则

- 顶级部门（公司根节点，level = 0）角色仅支持：**总经理（ceo）**、**副总经理（director）**
- 顶级部门允许多人，不再限制“只能一个人”

### 5.2 非顶层角色规则

- 非顶级部门角色仅支持：**负责人（manager）**、**普通员工（staff）**
- 同一部门只允许一个负责人

### 5.3 顶层人员兼任第二层负责人

- 总经理/副总经理可在人员表单中通过“兼任负责人”字段选择第二层部门
- 兼任后，该账号保留顶层角色，并额外具备对应第二层部门负责人权限
- 同一第二层部门同一时刻仅允许一名负责人（含“负责人角色用户”与“顶层兼任负责人”两种来源）

### 5.4 点击顶级节点展示所有人

- 点击左侧部门树的顶级节点，右侧加载**全部人员**
- 排序按照：总经理最优先 → 副总经理（按部门层级排列）→ 负责人（按部门层级排列）→ 普通员工（按部门层级排列）
- 此时分页不生效（一次加载全部数据）

## 六、新增人员

点击"新增人员"按钮弹出表单，包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 | 输入框 | 是 | |
| 密码 | 输入框 | 是 | 需满足密码强度要求，可点击"随机生成"自动填充 |
| 部门 | 树形选择 | 是 | 选择所属部门 |
| 职位 | 输入框 | 否 | |
| 手机号 | 输入框 | 是 | 同时作为登录账号 |
| 邮箱 | 输入框 | 否 | |
| 角色 | 下拉选择 | 是 | 顶级可选总经理/副总经理；其他层可选负责人/普通员工 |
| 兼任负责人 | 多选下拉 | 否 | 仅总经理/副总经理可选，用于兼任第二层部门负责人 |

### 密码强度要求

密码需同时包含：大写字母、小写字母、数字、特殊字符，长度不少于 8 位。

---

## 七、编辑人员

- 可选择是否重置密码（开关控制）
- 部门变更后角色选项按层级自动切换：顶层（总经理/副总经理）、其他层（负责人/普通员工）
- 同一部门只能有一个负责人（manager），保存时自动校验

---

## 八、删除人员

- 点击"删除"按钮后弹窗二次确认
- 即使该人员存在关联事务，也允许删除
- 删除实现为逻辑删除（status=0），以保留任务流程记录的外键引用；删除后不再出现在人员列表中

---

## 九、技术实现

### 9.1 前端

- 页面文件：`frontend/src/views/organization/MemberManageView.vue`
- 左侧树标题改为"部门结构"，并新增部门操作按钮（新增下级/编辑/删除）
- 树设置为默认展开且不可折叠
- `watch(form.deptId)`：部门变更时按层级切换角色可选项（顶层：总经理/副总经理；其他层：负责人/普通员工）
- 新增“兼任负责人”多选下拉：总经理/副总经理可选择兼任第二层部门负责人
- `sortedMemberList` computed：按角色优先级 + 部门层级 + 部门名 + 姓名排序
- 在姓名、部门、角色列增加 `sortable=custom`，支持点击箭头升降序排序
- 保存前校验同部门负责人唯一

### 9.2 后端

- 人员 CRUD：`MemberServiceImpl.java`
- 接口：`/api/member/list`（分页/条件查询）、`/api/member`（增删改）
- 人员新增/编辑时以手机号作为登录账号（username 字段自动与手机号保持一致）
- 人员删除不再拦截关联事务校验，改为逻辑删除（status=0）
- 人员列表与计数仅查询在职人员（status=1）

### 9.3 角色体系

| 角色代码 | 显示名称 | 可分配部门 | 限制 |
|----------|----------|------------|------|
| admin | 管理员 | 不可分配 | 固定账号，只有1个 |
| ceo | 总经理 | 仅顶级节点 | 可兼任第二层负责人 |
| director | 副总经理 | 仅顶级节点 | 可兼任第二层负责人 |
| manager | 负责人 | 非顶级节点 | 同一部门最多 1 人 |
| staff | 普通员工 | 非顶级节点 | — |
